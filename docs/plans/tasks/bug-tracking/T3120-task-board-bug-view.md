## Problem

The task board (launched via `/task-board`) shows tasks from PLAN.md but has no visibility into bug reports. Admin has to context-switch between the task board and email to see what needs attention. Bugs should be visible alongside tasks so the admin can prioritize work across both.

## Solution

Add a "Bugs" section to the task board that pulls from the `bug_reports` Postgres table via API. Show open bugs with status, reporter, mode, description preview, and timestamp. Click to expand full detail. Allow status updates (drag to resolve, mark as duplicate, etc.).

### Technical Notes

**Task board integration approach:**
The task board is a browser-based tool that reads PLAN.md. For bugs, it needs to additionally fetch from the backend API. Two options:

1. **API fetch from task board UI** - Task board JS fetches `/api/admin/bugs` and renders a bugs section. Requires the backend to be running.
2. **Skill pre-fetches and injects** - The task-board skill fetches bugs before launching the UI and injects them as data. Works offline but less dynamic.

Option 1 is better -- the task board already runs in a browser with the backend available.

**Bug card display:**
```
[NEW] Bug #42                              2026-05-24 01:35
Reporter: user@example.com
Mode: annotate | Game #7 | 18 clips
"Clip icon for 16 was placed in wrong part of timeline"
```

**Status columns (or filters):**
- New (unread)
- Investigating (someone's looking at it)
- Confirmed (real bug, needs fix)
- Resolved / Not a Bug / Duplicate (closed)

**Actions from task board:**
- Change status (drag between columns or dropdown)
- Mark as duplicate of another bug
- Add admin notes
- Promote to task (calls into T3130's promote flow)
